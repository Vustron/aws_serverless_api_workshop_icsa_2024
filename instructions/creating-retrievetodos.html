üîç Creating `getTodo`

Now, let's enhance our serverless application by creating a getTodo Lambda function. This function will retrieve a specific todo item by its id from our DynamoDB table using the AWS SDK v3.

üìú Step-by-Step: Defining the getTodo Lambda Function

1. Create the getTodo.ts File

  • What to do: We'll start by creating a new file for our getTodo Lambda function.

  • How:

    Open your terminal and run:

    touch lambda/getTodo.ts

    • This command creates a new TypeScript file named getTodo.ts in the lambda directory.

2. Write the Lambda Function Code

  • Open getTodo.ts in your code editor and add the following code:

    // Import necessary AWS SDK v3 clients and commands import { DynamoDBClient } from "@aws-sdk/client-dynamodb"; import { DynamoDBDocumentClient, GetCommand, GetCommandInput, } from "@aws-sdk/lib-dynamodb"; // Define the interface for a Todo item interface TodoItem { id: string; title: string; description: string; completed: boolean; } // Initialize the DynamoDB Client const ddbClient = new DynamoDBClient({ region: process.env.AWS_REGION, // Your AWS region }); // Initialize the DynamoDB Document Client const dynamoDb = DynamoDBDocumentClient.from(ddbClient); export const handler = async (event: any) => { try { const { id } = event.pathParameters; // Set up parameters for the Get operation const params: GetCommandInput = { TableName: process.env.TABLE_NAME!, // DynamoDB table name from environment variables Key: { id }, // The key of the item to retrieve }; // Execute the GetCommand to retrieve the item const data = await dynamoDb.send(new GetCommand(params)); if (data.Item) { return { statusCode: 200, body: JSON.stringify(data.Item), }; } else { return { statusCode: 404, body: JSON.stringify({ error: "Todo not found" }), }; } } catch (error) { console.error("Error retrieving todo item:", error); return { statusCode: 500, body: JSON.stringify({ error: "Could not retrieve the todo item" }), }; } };

3. Understanding the Code

Let's break down what this code does in simple terms.

a. Imports

  • AWS SDK v3 Modules:

    • DynamoDBClient: A client for interacting with DynamoDB at a low level.

    • DynamoDBDocumentClient, GetCommand, GetCommandInput: Higher-level abstractions to simplify our interactions with DynamoDB.

b. Define the TodoItem Interface

  • Purpose: Specifies the structure of a todo item for consistency and type safety.

  • Fields:

    • id: Unique identifier of the todo item.

    • title: Title of the todo.

    • description: Description of the todo.

    • completed: Boolean indicating if the todo is completed.

c. Initialize the DynamoDB Clients

  • ddbClient: Configured with your AWS region, it communicates with DynamoDB.

  • dynamoDb: Simplifies the process of converting data to and from DynamoDB format.

d. The Handler Function

  • Purpose: This function is executed when the Lambda function is triggered.

  • Process:

    1. Extract the id Parameter:

      • Retrieves the id from the URL path parameters.

    2. Set Up Parameters for DynamoDB:

      • Defines the table name and the key of the item to retrieve.

    3. Retrieve the Item:

      • Uses GetCommand to fetch the item from DynamoDB.

    4. Handle the Response:

      • If Item Exists: Returns a 200 status code with the item data.

      • If Item Not Found: Returns a 404 status code with an error message.

    5. Error Handling:

      • Logs any errors and returns a 500 status code.

‚ûï Adding getTodo to the CDK Stack

Now, let's integrate the getTodo Lambda function into our AWS CDK stack and connect it to API Gateway.

1. Define the getTodo Lambda Function in the Stack

  • Open your stack file (lib/serverless-app-stack.ts).

  • Add the following code:

    // Define the getTodo Lambda Function const getTodoFunction = new NodejsFunction(this, 'GetTodoFunction', { entry: path.join(__dirname, '../lambda/getTodo.ts'), // Path to your Lambda function code handler: 'handler', environment: { TABLE_NAME: table.tableName, // Passes the DynamoDB table name AWS_REGION: this.region, // Passes the AWS region }, });

2. Grant Read Permissions to the Lambda Function

  • Why? The function needs permission to read data from DynamoDB.

  • How:

    table.grantReadData(getTodoFunction);

3. Integrate with API Gateway

  • Add a GET method to the /todos/{id} resource to invoke getTodoFunction.

  • How:

    // Create a new resource for /todos/{id} const todo = todos.addResource('{id}'); // Integrate getTodo Lambda with GET method on /todos/{id} const getTodoIntegration = new apigateway.LambdaIntegration(getTodoFunction); todo.addMethod('GET', getTodoIntegration); // GET /todos/{id}

  • Explanation:

    • {id}: Defines a path parameter in the URL.

    • API Gateway Route: When a GET request is made to /todos/{id}, it triggers the getTodoFunction.

üß™ Testing and Deployment

1. Synthesize the CloudFormation Template

  • Command:

    cdk synth

  • Purpose: To verify the resources that will be created or updated.

2. Deploy the Stack

  • Command:

    cdk deploy

  • Note: Confirm the deployment when prompted.

3. Testing the GET Endpoint

a. Find Your API's URL

  • Where to look: The cdk deploy output or in the AWS Console under API Gateway.

b. Send a GET Request

  • Use Tools Like:

    • Postman

    • curl

  • Sample Request Using curl:

    curl https://your-api-id.execute-api.your-region.amazonaws.com/prod/todos/your-todo-id

    • Replace your-api-id, your-region, and your-todo-id with your actual API ID, region, and the ID of the todo item you want to retrieve.

  • Expected Responses:

    • Success (200): Returns the todo item in JSON format.

    • Not Found (404): Indicates the item does not exist.

    • Error (500): An internal server error occurred.

c. Verify the Data

  • Check in DynamoDB: Ensure the item retrieved matches the one in your TodoTable.
