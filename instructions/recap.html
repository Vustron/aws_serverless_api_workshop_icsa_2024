🏗️ Complete CDK Code: `createTodo` Reference

Now that we've gone through each part, here's the complete AWS CDK code to set up the createTodo Lambda function along with DynamoDB and API Gateway.

📜 Full Code: lib/serverless-app-stack.ts

import * as cdk from 'aws-cdk-lib'; import { Construct } from 'constructs'; import { NodejsFunction } from 'aws-cdk-lib/aws-lambda-nodejs'; import * as dynamodb from 'aws-cdk-lib/aws-dynamodb'; import * as apigateway from 'aws-cdk-lib/aws-apigateway'; import * as path from 'path'; export class ServerlessAppStack extends cdk.Stack { constructor(scope: Construct, id: string, props?: cdk.StackProps) { super(scope, id, props); // 🗄️ Create the DynamoDB Table const table = new dynamodb.Table(this, 'TodoTable', { partitionKey: { name: 'id', type: dynamodb.AttributeType.STRING }, tableName: 'TodoTable', billingMode: dynamodb.BillingMode.PAY_PER_REQUEST, }); // 🛠️ Define the createTodo Lambda Function const createTodoFunction = new NodejsFunction(this, 'CreateTodoFunction', { entry: path.join(__dirname, '../lambda/createTodo.ts'), // Path to your Lambda function code handler: 'handler', environment: { TABLE_NAME: table.tableName, // Pass the DynamoDB table name to the function }, }); // 🔒 Grant the Lambda function write permissions to the DynamoDB table table.grantWriteData(createTodoFunction); // 🌐 Set Up API Gateway REST API const api = new apigateway.RestApi(this, 'TodoApi', { restApiName: 'Todo Service', description: 'This service manages todos.', }); // 🗂️ Create the /todos Resource const todos = api.root.addResource('todos'); // 🔗 Integrate createTodo Lambda with POST method on /todos const createTodoIntegration = new apigateway.LambdaIntegration(createTodoFunction, { requestTemplates: { 'application/json': '{ "statusCode": "200" }' }, }); todos.addMethod('POST', createTodoIntegration); // POST /todos } }

🔍 Breaking Down the Code

Let's walk through the code step by step to understand how each part contributes to our serverless application.

1. Importing Necessary Modules

  • What is this? We import libraries that provide the tools we need to define AWS resources using CDK.

  • Modules Imported:

    • aws-cdk-lib: The core CDK library.

    • constructs: Provides the base Construct class for defining new components.

    • aws-lambda-nodejs: Helps in creating Node.js Lambda functions.

    • aws-dynamodb: Allows us to define DynamoDB tables.

    • aws-apigateway: Enables us to set up API Gateway.

    • path: A Node.js module for handling file paths.

2. Defining the CDK Stack

  • What is a Stack? Think of a stack as a container that holds all the AWS resources for your application.

  • In our code: We create a class ServerlessAppStack that extends cdk.Stack.

3. Creating the DynamoDB Table

const table = new dynamodb.Table(this, 'TodoTable', { partitionKey: { name: 'id', type: dynamodb.AttributeType.STRING }, tableName: 'TodoTable', billingMode: dynamodb.BillingMode.PAY_PER_REQUEST, });

  • Purpose: To store our todo items.

  • Key Components:

    • Partition Key (id): A unique identifier for each item.

    • Billing Mode: PAY_PER_REQUEST means we pay only for the read/write capacity we use.

4. Defining the createTodo Lambda Function

const createTodoFunction = new NodejsFunction(this, 'CreateTodoFunction', { entry: path.join(__dirname, '../lambda/createTodo.ts'), handler: 'handler', environment: { TABLE_NAME: table.tableName, }, });

  • Purpose: Handles the logic for creating new todo items.

  • Key Components:

    • Entry Point: Points to the code file for the Lambda function.

    • Handler: The function that gets executed when the Lambda is triggered.

    • Environment Variables: Passes the table name to the function for easy access.

5. Granting Permissions

table.grantWriteData(createTodoFunction);

  • Purpose: Allows the Lambda function to write data to the DynamoDB table.

  • Why is this important? Without this permission, the function cannot save new todo items to the table.

6. Setting Up API Gateway

const api = new apigateway.RestApi(this, 'TodoApi', { restApiName: 'Todo Service', description: 'This service manages todos.', });

  • Purpose: Provides an HTTP interface so clients can interact with our application.

  • Key Components:

    • RestApi Name and Description: Helps identify the API in AWS.

7. Creating the /todos Resource

const todos = api.root.addResource('todos');

  • Purpose: Defines a specific endpoint (/todos) for todo operations.

8. Integrating Lambda with API Gateway

const createTodoIntegration = new apigateway.LambdaIntegration(createTodoFunction, { requestTemplates: { 'application/json': '{ "statusCode": "200" }' }, }); todos.addMethod('POST', createTodoIntegration);

  • Purpose: Connects the createTodo Lambda function to the POST method on the /todos endpoint.

  • Key Components:

    • LambdaIntegration: Tells API Gateway to invoke our Lambda function when a request is made.

    • Method: Adds a POST method to the /todos resource.

🚀 Deploying and Testing Your Application

Now, let's get your application up and running!

1. Synthesize the CloudFormation Template

  • Command:

    cdk synth

  • What it does: Converts your CDK code into a CloudFormation template.

  • Why: Allows you to preview the AWS resources that will be created.

2. Deploy the Stack

  • Command:

    cdk deploy

  • What it does: Deploys your application to AWS.

  • Note: You may be prompted to confirm the deployment—type y and press Enter.

3. Testing the API Endpoint

a. Find Your API's URL

  • How to find it:

    • After deployment, the cdk deploy command will output the API endpoint URL.

    • Alternatively, go to the AWS Console, navigate to API Gateway, and find your API under TodoApi.

b. Send a POST Request

  • Use Tools Like:

    • Postman: A GUI application for sending HTTP requests.

    • curl: A command-line tool.

  • Sample Request Using curl:

    curl -X POST https://your-api-id.execute-api.your-region.amazonaws.com/prod/todos \ -H "Content-Type: application/json" \ -d '{ "id": "unique-id", "title": "Sample Todo", "description": "This is a sample todo item" }'

    • Replace your-api-id and your-region with your API's actual ID and region.

  • Expected Response:

    • Success: A message confirming that the todo item was created.

    • Failure: An error message indicating what went wrong.

c. Verify the Item in DynamoDB

  • Steps:

    • Go to the DynamoDB Console.

    • Find the TodoTable.

    • Check the items to see if your new todo item was added.
