Before we start building our serverless app, let's ensure your computer is set up with all the necessary tools.

üìå Required Tools

  1. Node.js (Version 14 or higher)

    • What is it? Node.js allows you to run JavaScript code on your computer outside of a web browser.

    • Why do we need it? We'll use it to run scripts and manage packages for our project.

    • Download it here: Node.js Official Site

  2. AWS CLI (Command Line Interface)

    • What is it? A tool to interact with AWS services directly from your terminal.

    • Why do we need it? To configure your AWS account and deploy resources.

    • Installation Guide: AWS CLI Installation

  3. AWS CDK (Cloud Development Kit)

    • What is it? A framework to define cloud infrastructure using familiar programming languages.

    • Why do we need it? It helps us write code to set up AWS services automatically.

    • How to install: Open your terminal and run:

      npm install -g aws-cdk

  4. Code Editor

    • What is it? An application where you write and edit your code.

    • Recommended Options:

      • Visual Studio Code (free and widely used)

      • IntelliJ IDEA (feature-rich, with student licenses available)

‚öôÔ∏è Setting Up AWS CLI with Your Account

  1. Configure AWS CLI with Your Credentials

    • Why? So your computer knows which AWS account to use when interacting with AWS services.

    • You'll need: AWS Access Key ID and Secret Access Key (provided to you).

    Steps:

    • Open your terminal and type:

      aws configure

    • When prompted, enter the following:

      • AWS Access Key ID: (We'll provide this to you)

      • AWS Secret Access Key: (We'll provide this to you)

      • Default region name: Choose a region close to you (e.g., us-west-2)

      • Default output format: Type json

    • Note: This AWS account has permissions to access services like Lambda, API Gateway, DynamoDB, etc.

  2. Verify Your Configuration

    • Why? To make sure everything is set up correctly.

    • How:

      Run in your terminal:

      aws sts get-caller-identity

      • Expected Result: You'll see your AWS account information, confirming the setup was successful.

üõ†Ô∏è Setting Up the CDK Project

  1. Create a New CDK Project

    • Why? To start a new AWS CDK application where we'll write our infrastructure code.

    • Steps:

      In your terminal, run:

      mkdir serverless-app cd serverless-app cdk init app --language typescript

      • Explanation:

        • mkdir serverless-app: Creates a new folder named serverless-app.

        • cd serverless-app: Enters the new folder.

        • cdk init app --language typescript: Initializes a new CDK project using TypeScript.

  2. Install Project Dependencies

    • Why? To add the AWS services we'll be using to your project.

    • Steps:

      Run:

      npm install @aws-cdk/aws-lambda @aws-cdk/aws-apigateway @aws-cdk/aws-dynamodb

      • What this does: Installs libraries for AWS Lambda (serverless functions), API Gateway (API endpoints), and DynamoDB (database).

‚úÖ Confirm Your Setup

Let's make sure everything is working before we proceed.

  1. Check CDK Installation

    • Run:

      cdk --version

    • Expected Result: The terminal displays the version number of AWS CDK installed.

  2. Verify AWS CLI Access

    • Run:

      aws sts get-caller-identity

    • Expected Result: Your AWS account details are displayed without any errors.

Great job! You're all set up to start building your serverless app. Next, we'll create our first AWS Lambda function.
