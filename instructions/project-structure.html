ğŸ§© Putting It All Together

Here's how these files and folders interact in our project:

  1. Starting the Application

    • Process: When you run cdk deploy, the CDK uses cdk.json to find the entry point (bin/serverless-app.ts), which in turn references the stack in lib/serverless-app-stack.ts.

    • Outcome: AWS resources are provisioned according to the definitions in your stack.

  2. Defining Infrastructure

    • Process: In lib/serverless-app-stack.ts, you write code to define AWS services.

    • Outcome: The CDK translates this code into CloudFormation templates, which AWS uses to create resources.

  3. Managing Dependencies

    • Process: package.json keeps track of libraries and tools your project needs.

    • Outcome: Ensures everyone working on the project has the same setup.

  4. Compiling Code

    • Process: tsconfig.json guides how TypeScript code is compiled.

    • Outcome: Your TypeScript code is converted to JavaScript that Node.js can execute.

  5. Testing Code

    • Process: Write tests in the test/ folder to validate your infrastructure.

    • Outcome: Catch errors early, ensuring your code works before deploying.
