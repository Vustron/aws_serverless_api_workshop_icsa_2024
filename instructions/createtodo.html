üöÄ Creating Lambda Functions

Now that we have our DynamoDB table set up, let's create a serverless function to interact with it. We'll start by creating a Lambda function called createTodo to add new items to our todo list.

üìù What is AWS Lambda?

  • AWS Lambda is a service that lets you run code without provisioning or managing servers.

  • Why use it? It allows you to focus on your application code, while AWS handles the infrastructure.

Analogy: Think of AWS Lambda as a chef in a restaurant who cooks your meal only when you place an order. There's no need to keep the chef on standby all day—you pay only when the service is used.

üìú Step-by-Step: Creating the createTodo Lambda Function

1. Create the createTodo.ts File

  • What to do: Create a new folder for your Lambda functions and add a file for createTodo.

  • How:

    Open your terminal and run:

    mkdir -p lambda touch lambda/createTodo.ts

    • mkdir -p lambda: Creates a new directory called lambda.

    • touch lambda/createTodo.ts: Creates a new TypeScript file for our function.

2. Write the Lambda Function Code

  • Open createTodo.ts in your code editor and add the following code:

    import { DynamoDBClient } from "@aws-sdk/client-dynamodb"; import { DynamoDBDocumentClient, PutCommand, PutCommandInput, } from "@aws-sdk/lib-dynamodb"; interface TodoItem { id: string; title: string; description: string; completed: boolean; } const ddbClient = new DynamoDBClient({ region: process.env.AWS_REGION, }); const dynamoDb = DynamoDBDocumentClient.from(ddbClient); export const handler = async (event: any) => { try { const data = JSON.parse(event.body); // Basic validation if (!data.id || !data.title || !data.description) { return { statusCode: 400, body: JSON.stringify({ error: "Missing required fields" }), }; } const todoItem: TodoItem = { id: data.id, title: data.title, description: data.description, completed: false, }; const params: PutCommandInput = { TableName: process.env.TABLE_NAME!, Item: todoItem, }; await dynamoDb.send(new PutCommand(params)); return { statusCode: 201, body: JSON.stringify({ message: "Todo created successfully!", item: todoItem, }), }; } catch (error) { console.error("Error creating todo item:", error); return { statusCode: 500, body: JSON.stringify({ error: "Could not create the todo item" }), }; } };

3. Understanding the Code

Let's break down what this code does in simple terms.

a. Imports

  • AWS SDK for JavaScript v3: We import necessary modules to interact with DynamoDB.

    • DynamoDBClient: Low-level client for DynamoDB operations.

    • DynamoDBDocumentClient, PutCommand, PutCommandInput: High-level abstractions to simplify our code.

b. Define the TodoItem Interface

  • What is an Interface? In TypeScript, an interface defines the shape of an object.

  • TodoItem: Specifies that every todo item will have:

    • id (string)

    • title (string)

    • description (string)

    • completed (boolean)

c. Initialize the DynamoDB Clients

  • ddbClient: Creates a new DynamoDB client, specifying the AWS region.

  • dynamoDb: Wraps ddbClient with DynamoDBDocumentClient to simplify data handling.

d. The Handler Function

  • Purpose: This is the function that AWS Lambda will execute when triggered.

  • Process:

    1. Parse the Incoming Data:

      • const data = JSON.parse(event.body);

      • Parses the JSON body of the incoming request.

    2. Validate the Data:

      • Checks if id, title, and description are present.

      • If any are missing, returns a 400 Bad Request response.

    3. Create the todoItem:

      • Constructs a new todo item object.

      • Sets completed to false by default.

    4. Prepare the Parameters for DynamoDB:

      • Defines the table name and the item to be inserted.

    5. Insert the Item into DynamoDB:

      • Uses PutCommand to add the item to the table.

    6. Return a Success Response:

      • Returns a 201 Created status with a success message.

  • Error Handling:

    • Catches any errors, logs them, and returns a 500 Internal Server Error.

‚ûï Adding the Lambda Function to the CDK Stack

Now, we'll tell AWS CDK about our new Lambda function so it can be deployed.

1. Install the Required CDK Module

  • Why? To work easily with Node.js Lambda functions in CDK.

  • How:

    Run in your terminal:

    npm install @aws-cdk/aws-lambda-nodejs

2. Import Necessary Modules

  • Open your stack file (lib/serverless-app-stack.ts).

  • Add these imports at the top:

    import { NodejsFunction } from '@aws-cdk/aws-lambda-nodejs'; import * as path from 'path';

3. Define the Lambda Function in the Stack

  • Add this code inside your stack class:

    const createTodoFunction = new NodejsFunction(this, 'CreateTodoFunction', { entry: path.join(__dirname, '../lambda/createTodo.ts'), // Path to your Lambda function code handler: 'handler', // The name of the exported function environment: { TABLE_NAME: table.tableName, // Passes the DynamoDB table name to the Lambda function }, });

  • Explanation:

    • NodejsFunction: A construct that simplifies working with Node.js Lambda functions.

    • entry: Specifies the path to the Lambda function code.

    • handler: The name of the function that's exported in your code (export const handler = ...).

    • environment: Environment variables available to the Lambda function. Here, we pass the DynamoDB table name.

4. Grant Permissions to the Lambda Function

  • Why? The Lambda function needs permission to write to the DynamoDB table.

  • How:

    Add this line after defining the Lambda function:

    table.grantWriteData(createTodoFunction);

üß™ Testing and Deploying

1. Synthesize the CloudFormation Template

  • What is this? Converts your CDK code into a CloudFormation template.

  • How:

    Run in your terminal:

    cdk synth

  • Why? To review the resources that will be created and catch any errors.

2. Deploy the Stack

  • How:

    cdk deploy

  • What happens? CDK will deploy your DynamoDB table and Lambda function to AWS.

  • Note: You might be prompted to approve the changes. Type y and press Enter.

‚úÖ Verifying Your Lambda Function

1. Check in the AWS Management Console

  • Go to the Lambda Console:

    AWS Lambda Console

  • Find Your Function:

    • Look for CreateTodoFunction in the list of functions.

  • Review the Configuration:

    • Code Source: Verify that your code is present.

    • Environment Variables: Check that TABLE_NAME is set correctly.

    • Permissions: Ensure the function has the necessary roles and policies.

2. Test the Function Manually

  • Create a Test Event:

    • In the Lambda console, click on "Test".

    • Choose "Create new test event".

    • Enter a name for the event (e.g., testCreateTodo).

    • Paste the following JSON into the event body:

      { "body": "{\"id\": \"1\", \"title\": \"Test Todo\", \"description\": \"This is a test todo item.\"}" }

  • Run the Test:

    • Click "Test" to execute the function.

  • Check the Result:

    • Success: You should see a 201 status code and a success message.

    • Verify in DynamoDB:

      • Go to the DynamoDB console and check if the new item was added to TodoTable.
