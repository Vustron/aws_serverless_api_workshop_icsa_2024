ğŸ’¡ Why Go Serverless?

Imagine if you could run your applications without having to set up or manage any servers. Serverless architecture makes this possible by letting cloud providers like AWS handle the server management for you.

Benefits:

  • Automatic Scalability: Your application can handle more users automatically without extra work.

  • Cost-Efficient: Pay only for the compute time you use—no need to maintain idle servers.

  • Focus on Code: Spend more time developing features and less time on infrastructure.

Key Components:

  • AWS Lambda: Think of this as your code running in the cloud, triggered by events, without any server setup.

  • API Gateway: This is like a receptionist for your application, directing incoming requests to the right place.

  • DynamoDB: A flexible, fast database service where you can store and retrieve data efficiently.

ğŸ›  Getting Started

Before the workshop, please:

  • Create an AWS Account

    • Sign up at aws.amazon.com/free.

  • Install Node.js and npm

    • Download from nodejs.org.

  • Set Up AWS CLI

    • Install instructions at AWS CLI User Guide.

  • Install AWS CDK

    • Open your terminal and run:

      npm install -g aws-cdk

  • Choose a Code Editor

    • Recommended: Visual Studio Code.

Step-by-Step Setup Guide

  1. Verify Node.js and npm Installation

    Open your terminal and type:

    node -v npm -v

    If you see version numbers, you're good to go!

  2. Configure AWS CLI with Your Credentials

    In the terminal, run:

    aws configure

    You'll be prompted to enter your AWS Access Key ID and Secret Access Key, which you can find in your AWS account.

  3. Check AWS CDK Installation

    Confirm that AWS CDK is installed by running:

    cdk --version
