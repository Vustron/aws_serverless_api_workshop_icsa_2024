Now, let's complete our serverless application by adding the deleteTodo Lambda function. This function will allow us to delete a specific todo item by its id from our DynamoDB table using the AWS SDK v3.

üìú Step-by-Step: Defining the deleteTodo Lambda Function

1. Create the deleteTodo.ts File

  • What to do: Create a new file for the deleteTodo Lambda function.

  • How:

    Open your terminal and run:

    touch lambda/deleteTodo.ts

    • This command creates a new TypeScript file named deleteTodo.ts in the lambda directory.

2. Write the Lambda Function Code

  • Open deleteTodo.ts in your code editor and add the following code:

    // Import necessary AWS SDK v3 clients and commands import { DynamoDBClient } from "@aws-sdk/client-dynamodb"; import { DynamoDBDocumentClient, DeleteCommand, DeleteCommandInput, } from "@aws-sdk/lib-dynamodb"; // Initialize the DynamoDB Client const ddbClient = new DynamoDBClient({ region: process.env.AWS_REGION, // Your AWS region }); // Initialize the DynamoDB Document Client const dynamoDb = DynamoDBDocumentClient.from(ddbClient); export const handler = async (event: any) => { try { const { id } = event.pathParameters; // Set up parameters for the Delete operation const params: DeleteCommandInput = { TableName: process.env.TABLE_NAME!, // DynamoDB table name from environment variables Key: { id }, // The key of the item to delete }; // Execute the DeleteCommand to remove the item await dynamoDb.send(new DeleteCommand(params)); // Return a successful response return { statusCode: 200, body: JSON.stringify({ message: "Todo deleted successfully" }), }; } catch (error) { console.error("Error deleting todo item:", error); // Return an error response return { statusCode: 500, body: JSON.stringify({ error: "Could not delete the todo item" }), }; } };

3. Understanding the Code

Let's break down what this code does in simple terms.

a. Imports

  • AWS SDK v3 Modules:

    • DynamoDBClient: A client for interacting with DynamoDB at a low level.

    • DynamoDBDocumentClient, DeleteCommand, DeleteCommandInput: Higher-level abstractions that simplify our interactions with DynamoDB.

b. Initialize the DynamoDB Clients

  • ddbClient: Configured with your AWS region, it communicates with DynamoDB.

  • dynamoDb: Simplifies the process of converting data to and from DynamoDB format.

c. The Handler Function

  • Purpose: This function is executed when the Lambda function is triggered.

  • Process:

    1. Extract the id Parameter:

      • Retrieves the id from the URL path parameters.

    2. Set Up Parameters for DynamoDB:

      • Defines the table name and the key of the item to delete.

    3. Delete the Item:

      • Uses DeleteCommand to remove the item from DynamoDB.

    4. Handle the Response:

      • On success, returns a 200 OK status with a success message.

      • On failure, logs the error and returns a 500 Internal Server Error.

‚ûï Adding deleteTodo to the CDK Stack

Now, let's integrate the deleteTodo Lambda function into our AWS CDK stack and connect it to API Gateway.

1. Define the deleteTodo Lambda Function in the Stack

  • Open your stack file (lib/serverless-app-stack.ts).

  • Add the following code:

    // Define the deleteTodo Lambda Function const deleteTodoFunction = new NodejsFunction(this, 'DeleteTodoFunction', { entry: path.join(__dirname, '../lambda/deleteTodo.ts'), handler: 'handler', environment: { TABLE_NAME: table.tableName, AWS_REGION: this.region, }, });

2. Grant Write Permissions to the Lambda Function

  • Why? The function needs permission to delete data in DynamoDB.

  • How:

    table.grantWriteData(deleteTodoFunction);

üåê Integrate with API Gateway

To make the deleteTodo function accessible via HTTP requests, we'll add a DELETE method to the /todos/{id} endpoint in API Gateway.

1. Add the DELETE Method to /todos/{id}

  • How:

    // Ensure you've already added the '{id}' resource under '/todos' const todo = todos.addResource('{id}'); // Integrate deleteTodo Lambda with DELETE method on /todos/{id} const deleteTodoIntegration = new apigateway.LambdaIntegration(deleteTodoFunction); todo.addMethod('DELETE', deleteTodoIntegration); // DELETE /todos/{id}

üß™ Testing and Deployment

1. Synthesize the CloudFormation Template

  • Command:

    cdk synth

  • Purpose: To verify the resources that will be created or updated.

2. Deploy the Stack

  • Command:

    cdk deploy

  • Note: Confirm the deployment when prompted.

3. Testing the DELETE Endpoint

a. Find Your API's URL

  • Where to look: The cdk deploy output or in the AWS Console under API Gateway.

b. Send a DELETE Request

  • Use Tools Like:

    • Postman

    • curl

  • Sample Request Using curl:

    curl -X DELETE https://your-api-id.execute-api.your-region.amazonaws.com/prod/todos/your-todo-id

    • Replace your-api-id, your-region, and your-todo-id with your actual API ID, region, and the ID of the todo item you want to delete.

  • Expected Response:

    • Success (200): Returns a message indicating the todo was deleted successfully.

    • Error (500): An internal server error occurred.

c. Verify the Deletion

  • Check in DynamoDB:

    • Go to the DynamoDB Console.

    • Find the TodoTable.

    • Verify that the item has been removed.
